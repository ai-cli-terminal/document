# 07. 요구사항 정의서
> **프로젝트명**: AI CLI 통합 리눅스 터미널
> **버전**: v1.0
> **작성일**: 2026-06-01
> **기술 스택**: Rust · ratatui · tokio · portable-pty · SQLite (대안: Go)
---

> 설계 철학: **"AI는 명령 실행자가 아니라 의사결정 보조자"**. 기본 동작은 일반 터미널과 동일하게 유지하고,
> AI 기능은 명시적 호출·실행 전 설명·위험도 분석·사용자 확인을 중심으로 제공한다.
> 기능 요구 ID: `FR-<AREA>-<NNN>` (AREA ∈ SHELL/AI/SEC/CTX/GUARD/STORE/SKILL/MCP/REMOTE/OPS).
> 비기능 요구 ID: `NFR-<AREA>-<NNN>` (AREA ∈ PERF/SEC/REL/COMPAT/PRIV/A11Y/OPS).
> 우선순위: **P0**(MVP+) / **P1**(Phase 2) / **P2**(Phase 3+).
> "관련 화면" 열의 ID는 `06_화면_기능_정의서.md`의 인벤토리 ID와 일치한다.

## 1. 이해관계자 요구 요약

| 이해관계자(페르소나) | 핵심 요구 | 우선 가치 |
|---|---|---|
| DEV (개발자) | 자연어로 명령 생성·설명·디버깅, preview/diff로 안전 확인, 프로젝트 컨텍스트 기반 추천 | 생산성 + 안전 |
| OPS (운영자·SRE) | 원격/컨테이너/k8s 컨텍스트 명확화, 로그·에러 분석, 위험 명령 차단, 감사 로그 | 안정성 + 추적성 |
| RESEARCH (연구자) | 반복 작업 자동화 스크립트, 로그 요약, 세션 재현, 오프라인/로컬 우선 동작 | 재현성 + 자율성 |
| SECADMIN (보안·조직 관리자) | Secret/PII 마스킹, 정책 프로파일·조직 정책, 원격 승인 통제, 마스킹 실패 fail-closed | 보안 + 거버넌스 |

공통 요구(설계 원칙 17개 기반):

- 일반 쉘 호환성이 AI 기능보다 우선한다. **AI 장애가 터미널 장애로 전파되지 않아야 한다.**
- PTY 상태와 AI 컨텍스트는 항상 동기화된다. 로컬·SSH·컨테이너·k8s 컨텍스트는 명확히 구분한다.
- 모든 외부 컨텍스트는 Zero-Trust Pipeline을 통과하며, Secret/PII 마스킹은 원격 호출 전 필수다.
- AI 생성 명령은 기본 자동 실행하지 않으며, 파일 변경 명령은 preview·dry-run·diff를 우선 제공한다.
- 정적 정책 검사(실행 전)와 동적 가드레일(실행 중)을 함께 사용한다.

> → 설계 목표 `00-overview-architecture.md` §2, 17개 원칙 §3 참조. MVP+ 범위는 §25.1·§30·§31.

## 4. 제약사항

| 구분 | 제약 | 근거 |
|---|---|---|
| 플랫폼 capability 편차 | guardrails(cgroups·inotify·seccomp·eBPF)·샌드박스 백엔드는 Linux/WSL/macOS별 가용성이 다르다. capability matrix로 가용 기능만 활성화한다. | §8.4, §31 |
| 로컬 우선 | 위험도 판단·정책 평가는 로컬에서 먼저 수행한다. 민감 컨텍스트는 원격 전송 대신 로컬 AI를 강제한다. | 원칙 9, §29.10 |
| AI 실패 비전파 | AI 기능 장애는 터미널 장애로 전파되지 않아야 한다. 오프라인/제한 네트워크에서도 일반 터미널 기능은 유지된다. | 원칙 3, §16.2 |
| 샌드박스 한계 | 샌드박스는 보조 안전장치이지 완전한 보안 경계가 아니다. 샌드박스 성공이 호스트 안전을 보장하지 않으며, 실패 시 자동 실제 실행 금지. | 원칙 13, §6.3 |
| 정규식 마스킹 한계 | 정규식 기반 마스킹은 완전하지 않다. 엔트로피 기반 시크릿 탐지를 보강하고, 탐지 거부 시 fail-closed. | §10.4, §29.8 |
| LLM 비결정성 | LLM 출력은 비결정적이므로 정확 일치가 아닌 LLM-as-judge·속성 기반 평가를 사용한다. | §22.6, §29.13 |
| 원격 승인 통제 | 원격 제어는 명시적 페어링·최소 권한·E2E 기본. 원격 승인은 Low/Medium까지 opt-in, **High·Critical 명령·외부 부작용 MCP 도구는 기본 차단**(로컬 승인). | 원칙 20, §28.4, §30-13 |
| MVP 범위 제외 | 데몬·MCP·원격 릴레이·조직 정책 배포·로컬 LLM·시맨틱 인덱싱은 MVP에서 제외(선택/후속 Phase). | §30, §32 |
