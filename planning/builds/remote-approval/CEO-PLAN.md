### PWA 인터랙션 상태 명세 (plan-design-review 2026-06-02 — Cockpit 문법)
C v2 목업은 "승인 대기(happy)" 상태. 나머지 상태는 같은 Cockpit 문법(패널·상태 라이트·게이지)으로:

| 상태 | 사용자가 보는 것 |
|---|---|
| 승인 대기(happy) | C v2 목업 (게이지+명령+머신 상태+분해+거부/승인) |
| LOADING(복호화) | 게이지 자리에 흐린 스켈레톤 + "복호화 중", 버튼 비활성 |
| EMPTY(대기 없음) | 카드 대신 차분한 빈 상태: "대기 중인 승인 없음" + 데몬 상태 라이트만 (warmth, 경보 아님) |
| ERROR(복호/네트워크/만료 nonce) | 적색 상태 라이트 + 이유("복호화 실패"/"연결 끊김"/"요청 만료") + 복구 액션("재연결"/"닫기"), 승인 버튼 없음 |
| STALE 경고 | 게이지 위 amber 오버레이 배너 "컨텍스트 변경됨 — 재확인 필요", 승인 버튼은 "그래도 승인"으로 약화 또는 보류 |
| Critical=local-only | 게이지 적색 + 버튼 자리에 "원격 승인 불가 — 로컬 터미널에서 승인하세요"(§30-13), 승인 버튼 제거 |
| 데몬 오프라인(#2) | 카드 dim + 상단 경고 "데몬 도달 불가 / 상태 불명 — 가드레일이 꺼져 있을 수 있음", 버튼 비활성 |
| 승인/거부 후 SUCCESS | 큰 체크/엑스 + "승인됨 — 머신에서 실행" / "거부됨 — 차단", 2~3초 후 EMPTY로 |
| WebAuthn 게이트 | Approve 탭 → OS 생체인증 시트(시스템 제공) → 성공 시 서명 회신, 실패 시 카드 유지 + "인증 실패, 재시도" |

- 그려진 목업은 happy 상태만(키 없어 추가 AI 생성 불가). 위 스펙이 구현 기준.

- **색 비의존**: 상태 라이트·위험 등급은 색+텍스트+아이콘 중복(§29.14). 적색 상태도 텍스트 라벨 동반.
- **터치 타깃 ≥44px**: 거부/승인 버튼 확인.
