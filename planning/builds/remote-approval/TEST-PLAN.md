## Affected Pages/Routes (PWA)
- `/approve` (승인 뷰) — payload 복호화 → 컨텍스트(+stale 경고)+위험도 0~100 표시 → WebAuthn 게이트 → Approve/Reject → 서명 토큰 회신
- `/pair` (페어링 뷰) — QR 스캔 → 디바이스 키쌍 생성 → 공개키 등록

## Key Interactions to Verify
- arm/disarm: `ai remote arm` 후에만 원격 승인 개입, 평소엔 로컬 확인 (셸 비블로킹)
- 승인 한 바퀴: armed + 위험 명령 → 폰 진동 → Approve → TOCTOU 재체크 통과 → 실행
- WebAuthn 게이트: 승인 누르기 전 기기 잠금/생체인증 통과 요구
- 거부/편집: Reject 시 명령 차단, Edit 시 수정안 회신

## Edge Cases
- 폰 오프라인/timeout → fail-closed(차단 + 로컬 승인 안내), 셸 무한 대기 없음
- 데몬 다운 → hook no-op, 일반 셸 정상 동작 (원칙 1·3)
- TOCTOU 드리프트: 승인 후 cwd/branch 변경 → 실행 직전 context_hash 재체크 → 불일치면 거부+재승인
- replay: 동일 승인 토큰 재전송 → nonce 1회용·expires_at 만료로 거부
- revoke: `ai remote revoke <device>` 후 그 디바이스 서명 토큰 거부
- Critical 명령: 어떤 경우에도 원격 승인 불가 (거부+로컬 안내)
- High 명령: 기본 차단, opt-in 오버라이드 시에만 원격 승인 (§30-13)
- stale 컨텍스트(cwd≠pwd / branch 변경 / 5분 경과): payload에 stale 경고 또는 보류
- 마스킹: payload의 cwd/git/명령어에 Secret/PII 마스킹 적용 (private key 차단)

## Critical Paths (보안-핵심, ship 게이트 — proptest + known-answer + 음성 케이스)
- 크립토 프로토콜: handshake → AEAD payload → 서명 → 검증. KAT로 "암호문만"·"재사용 거부" 증명
- 위험도 점수 0~100: deterministic — 동일 명령·환경 동일 점수 (proptest + golden set, shared-core 크레이트)
- 위험도 게이트: Low/Med opt-in, High opt-in 오버라이드, Critical 절대 차단 (등급 × balanced/paranoid)
- 마스킹 baseline §31.8: 룰별 + private key block → 원격 차단
- revoke / replay / TOCTOU 거부: 세 보안 음성 케이스 모두 통과 필수
