# 15. 사용자 메뉴얼
> **프로젝트명**: AI CLI 통합 리눅스 터미널
> **버전**: v1.0
> **작성일**: 2026-06-01
> **기술 스택**: Rust · ratatui · tokio · portable-pty · SQLite (대안: Go)
---

> 이 메뉴얼은 최종 사용자(개발자·운영자, 일부 비개발자 포함)를 위한 안내서입니다.
> 핵심 철학: **AI는 의사결정을 돕는 보조자이며, 최종 실행 결정은 항상 사용자가 합니다.**
> AI가 명령을 만들어 주더라도, 보여 주고 → 확인받고 → 사용자가 허락해야 실행됩니다.
> 화면 모양 예시는 `→ 06_화면_기능_정의서.md 참조`.

모든 기능은 다음 3단계로 동작합니다.

```text
① 입력  →  ② 확인  →  ③ 실행
(무엇을 시킬지)  (AI 제안과 위험도 보기)  (사용자가 Enter로 허락)
```

## 1. 시작하기

### 1.1 설치

이 도구는 평소 쓰던 리눅스 터미널을 그대로 감싸 동작합니다. Linux를 우선 지원하며 WSL과 macOS에서도 사용할 수 있습니다(일부 안전 기능 차이는 §7 참조).

- 설치하면 `ai` 명령을 쓸 수 있게 됩니다.
- 만약 시스템에 이미 `ai`라는 이름의 다른 프로그램이 있으면, 충돌을 감지해 알려 줍니다. 이때 `aish` 같은 대체 이름을 쓰도록 안내합니다.

```text
Command name 'ai' already exists at /usr/local/bin/ai.
Use alternative trigger 'aish'? [Y/n]
```

### 1.2 셸 통합 (`ai init shell`)

`cd`로 폴더를 옮기거나 `git`으로 브랜치를 바꾸는 등 터미널 상태가 바뀌었을 때, AI가 그 변화를 정확히 알도록 셸과 연결하는 단계입니다. 이 연결을 셸 통합이라고 합니다.

중요: 이 도구는 **여러분의 셸 설정 파일(`~/.bashrc`, `~/.zshrc`)을 마음대로 고치지 않습니다.** 아래 명령을 직접 실행했을 때만, 그것도 미리 보여 주고 동의를 받은 뒤에만 변경합니다. (MVP 지원 셸: `bash`, `zsh`)

```bash
ai init shell             # ① 입력: 셸에 연결 (변경 내용 확인 후 동의)
ai init shell --dry-run   # 실제로 바꾸지 않고 미리보기만
ai init shell --diff      # 추가될 내용을 diff로 표시
ai init shell --uninstall # 이 도구가 넣은 줄만 제거 (직접 쓴 줄은 건드리지 않음)
```

설치 시 추가되는 내용을 먼저 보여 줍니다.

```diff
--- ~/.zshrc
+++ ~/.zshrc
@@ -42,3 +42,8 @@
 export PATH="$HOME/.local/bin:$PATH"
+# AI Terminal integration
+if command -v ai >/dev/null 2>&1; then
+  eval "$(ai shell-hook zsh)"
+fi
```

`Apply? [y/N]`에서 `y`를 눌러야 적용됩니다. 마음에 들지 않으면 `--uninstall`로 깔끔하게 되돌릴 수 있습니다.

> 셸 통합이 안 되어 있어도 터미널과 AI 기능은 정상 동작합니다. 다만 폴더 이동 같은 상태 변화 인식이 조금 늦어질 수 있어 통합을 권장합니다.

### 1.3 첫 실행

설치 후 터미널을 새로 열면 평소처럼 명령을 입력할 수 있습니다. 처음 한 번 진단을 돌려 보면 준비 상태를 확인할 수 있습니다.

```bash
ai doctor
```

```text
AI Terminal Doctor
Shell:   bash 5.2 detected
PTY:     OK
AI provider: remote provider reachable
Policy:  active profile: balanced
Masking: secret rules 18 loaded / pii rules 7 loaded
Context: cwd synchronized / git branch synchronized
```

### 1.4 정책 프로파일 선택

정책 프로파일은 "AI 명령을 얼마나 까다롭게 확인할지" 정하는 안전 수준입니다. MVP에는 두 가지가 있습니다.

| 프로파일 | 한 줄 설명 | 누구에게 |
|---|---|---|
| `balanced` (기본) | 위험한 명령만 확인, 일반 명령은 매끄럽게 | 대부분의 사용자 |
| `paranoid` | 모든 AI 명령을 확인, 원격 AI 차단, 강한 마스킹 | 민감 환경·보안 우선 |

```bash
ai policy show              # ① 현재 정책 보기
ai policy set paranoid      # ② 더 엄격하게 전환 (즉시 반영)
```

두 프로파일 모두 시스템을 손상시킬 수 있는 **Critical 명령은 항상 차단**합니다.
(보안 관리자가 더 세분화된 프로파일을 운영할 경우 `→ 16_운영_메뉴얼.md 참조`)

## 2. 기본 사용

### 2.1 일반 터미널은 그대로 동작합니다

가장 중요한 약속: **평소 쓰던 명령은 전혀 바뀌지 않습니다.** AI를 부르지 않으면 일반 리눅스 터미널과 100% 동일하게 동작합니다.

```bash
cat app.log | grep ERROR | less
vim ~/.bashrc
top
ssh user@example.com
docker exec -it app bash
```

파이프, 리다이렉션, 백그라운드 작업, `vim`·`htop` 같은 대화형 프로그램, `Ctrl+C`/`Ctrl+D`/`Ctrl+Z` 모두 그대로 됩니다.

### 2.2 자연어로 명령 시키기 (`ai "..."`)

무엇을 하고 싶은지 한국어(또는 영어)로 말하면, AI가 적절한 명령을 만들어 줍니다. 핵심은 **AI가 바로 실행하지 않고, 먼저 보여 준다**는 점입니다.

```text
① 입력
$ ai "현재 폴더에서 7일 지난 .log 파일 찾아줘"

② 확인 (AI가 만든 명령과 위험도를 보여 줌)
Goal: 7일 이상 된 로그 파일 조회
Command:
  find . -type f -name "*.log" -mtime +7
Risk: Low — 파일을 삭제하지 않고 조회만 합니다.
Next: 실행 Enter / 수정 e / 취소 Ctrl+C

③ 실행
(내용을 읽고 괜찮으면 Enter, 고치고 싶으면 e, 그만두려면 Ctrl+C)
```

주요 입력 방법:

| 무엇을 원하나 | 명령 | 예시 |
|---|---|---|
| 질문에 답을 듣고 싶다 | `ai ask` | `ai ask "열려 있는 포트 확인 방법"` |
| 명령만 만들어 보고 싶다 | `ai cmd` | `ai cmd "node_modules 삭제"` |
| 만든 명령을 확인 후 실행 | `ai run` | `ai run "console.log 제거해줘"` |
| 실행 전 변경 결과를 미리 보고 싶다 | `ai preview` | `ai preview "foo를 bar로 치환"` |

### 2.3 실행 전 확인 · 위험도 읽는 법

명령을 실행하기 전, 화면에는 항상 **위험도(Risk)** 가 표시됩니다. 색뿐 아니라 글자 라벨과 아이콘으로도 표시하므로 색 구분이 어려워도 읽을 수 있습니다(`[HIGH] ⚠`).

| 위험도 | 점수 | 뜻 | 화면 동작 |
|---|---|---|---|
| Low | 0~24 | 읽기 위주, 부작용 없음 | 보통 바로 보여 줌 |
| Medium | 25~49 | 일부 파일/프로세스 변경 | 확인 요청 |
| High | 50~79 | 광범위 변경·권한 상승·네트워크 | 강한 확인 + 미리보기 |
| Critical | 80~100 | 시스템 손상·대량 삭제 위험 | **차단** |

위험한 이유도 함께 설명합니다.

```text
Risk: High (score 5)
+3 recursive delete, +3 wildcard target, -1 scoped to ./build
Affected: ~1,240 files under ./build
```

가끔 위험도는 낮아도 "AI가 확신하지 못함" 배지가 붙을 수 있습니다. 요청이 모호해 AI가 추측한 경우이니, 한 번 더 읽고 결정하세요.

```text
Generated command: rm ./*.tmp
Confidence: Low — "임시 파일"의 범위를 *.tmp로 가정했습니다. 다른 확장자도 포함할까요?
Proceed with this assumption? [y/N/edit]
```

### 2.4 설명과 수정 (`ai explain` · `ai fix`)

- **`ai explain`**: 어떤 명령이나 출력이 무슨 뜻인지 풀어서 설명해 줍니다.

```bash
ai explain "ps aux | grep nginx"
```

- **`ai fix`**: 방금 실패한 명령의 원인을 분석하고, 고친 명령을 **제안**합니다. (자동으로 다시 실행하지는 않습니다 — 확인은 사용자 몫입니다.)

```text
$ ai fix last-error
The AI-generated command failed.
Cause: The target directory does not exist.
Previous: tar -czf backup.tar.gz ./data
Suggested fix: mkdir -p ./data && tar -czf backup.tar.gz ./data
Risk: Medium — Creates a directory and archives it.
Run suggested fix? [y/N]
```

## 3. 역할별 가이드

### 3.1 DEV — 개발자

- **명령 생성**: `ai cmd "현재 브랜치에서 변경된 파일만 stash"` 처럼 자연어로 명령을 만듭니다.
- **에러 분석**: 빌드·테스트가 깨지면 `ai fix last-error`로 원인과 수정안을 받습니다.
- **스크립트 생성**: `ai script "매일 새벽 로그 압축"`으로 반복 작업 스크립트를 생성합니다. 생성된 스크립트도 실행 전 미리보기 대상입니다.
- **코드 수정 미리보기**: `ai run "src의 console.log 전부 제거"`는 실행 전 diff를 보여 주고, 적용 후에는 잠시 동안 `ai undo`로 되돌릴 수 있습니다(§4.2).

### 3.2 OPS — 운영자 (SRE)

- **로그 요약**: `ai summarize /var/log/nginx/error.log`로 긴 로그의 핵심만 빠르게 봅니다(큰 로그는 단계적으로 요약, 최대 180초).
- **서비스 점검**: `ai ask "nginx가 떠 있는지, 포트가 열렸는지 확인"`처럼 점검 방법을 물어볼 수 있습니다. 서비스 재시작 같은 명령은 위험도 High로 강한 확인을 거칩니다.
- **원격 컨텍스트**: SSH·컨테이너에 들어가면 프롬프트가 현재 위치를 분명히 표시해, 로컬과 원격을 헷갈리지 않게 합니다.

```text
[local:/home/user/project]$
[ssh:prod-web-01:/var/www/app]$
[container:app:/usr/src/app]$
```

### 3.3 RESEARCH — 연구자

- **데이터·로그 분석**: `ai summarize`로 대용량 결과 파일을 요약하고, `ai ask`로 분석 방법을 묻습니다.
- 분석 대상 파일에 적힌 문구(예: "이전 지시를 무시하라")는 명령이 아니라 **데이터로만** 취급되므로 안심하고 쓸 수 있습니다.

### 3.4 SECADMIN — 보안 · 조직 관리자

- **엄격 모드 사용**: `ai policy set paranoid`로 모든 AI 명령을 확인하고 원격 AI를 차단합니다.
- **마스킹 확인**: 원격 AI로 무엇이 전송되는지 실행 전에 확인할 수 있습니다. 민감 정보는 자동으로 가려집니다.

```text
The following context will be sent to the remote AI provider:
- User request / Current directory path / Last command and exit code
- 120 lines of stderr / Git branch name / package.json scripts
Masked: 3 email addresses / 1 API token / 2 IP addresses
Send? [Y/n]
```

> 조직 배포·감사·보존 정책 등 관리 업무는 `→ 16_운영_메뉴얼.md 참조`.

## 4. 안전 기능 이해

### 4.1 미리보기와 diff (preview / diff)

파일을 바꾸는 명령은 **실행 전에 무엇이 바뀌는지** 보여 줍니다. 가능한 경우 원본을 건드리지 않고 임시 복사본에서 먼저 돌려 결과를 비교합니다.

```text
Proposed command: sed -i 's/foo/bar/g' config.txt
Preview diff:
--- config.txt
+++ config.txt
-foo_enabled=true
+bar_enabled=true
Risk level: Medium   Apply changes? [y/N]
```

`rm` 같은 삭제 명령은 지울 파일 목록과 개수를 먼저 보여 줍니다. 외부 시스템 변경·DB 마이그레이션처럼 미리 볼 수 없는 명령은 그 이유를 분명히 알려 줍니다.

### 4.2 되돌리기 (`ai undo`)

파일 수정 같은 일부 명령은 실행 전 자동 백업을 만들어, 실행 후에도 일정 기간(기본 7일) 안에 되돌릴 수 있습니다.

```text
$ ai run "src의 console.log 전부 제거"
... applied (txn: 7f3a). Undo available.
$ ai undo last
Reverting txn 7f3a: 3 files restored from backup.
```

주의: `dd`, `mkfs`, 패키지 설치/삭제, 서비스 재시작, 네트워크 전송 등 **되돌릴 수 없는 작업은 undo 대상이 아닙니다.** 이 경우 실행 전에 "되돌릴 수 없습니다"라고 명시합니다.

### 4.3 민감 정보 마스킹

API 키·토큰·비밀번호·이메일·IP·주민번호 등 민감 정보는 AI에 보내기 전, 그리고 기록을 저장하기 전 자동으로 가려집니다. 원문 비밀값은 디스크에 저장되지 않습니다. 비밀번호 입력 화면이 감지되면 AI는 그 내용을 수집하지 않습니다.

### 4.4 Critical 명령 차단

`rm -rf /`, `mkfs`, `dd if=... of=/dev/sda` 같은 치명적 명령은 어떤 프로파일에서도 실행되지 않고 차단됩니다. 또한 실행 도중이라도 비정상 행동(대량 삭제 등)이 감지되면 멈추고 물어봅니다.

```text
Execution paused. The command is attempting to delete more than 1,000 files.
Command: rm -rf ./cache   Affected path: ./cache   Continue? [y/N]
```

### 4.5 취소 (Ctrl+C)

AI 응답을 기다리다 `Ctrl+C`를 누르면 **터미널은 꺼지지 않고** 그 AI 요청만 취소된 뒤 바로 프롬프트로 돌아옵니다. 네트워크가 느리거나 응답이 지연돼도 터미널이 멈추지 않습니다.

## 5. 토큰 · 비용 · 예산 보기 (`/usage`)

AI 사용량과 예상 비용을 언제든 확인할 수 있습니다. AI 전용 모드(`ai shell`) 안에서는 `/usage`로 봅니다.

```text
AI usage: 3,240 input / 620 output tokens   Estimated cost: $0.0041
Session total: $0.083 / $2.00   Monthly total: $12.42 / $30.00
```

- 기본 예산: 세션 $2 / 월 $30.
- 예산의 **80%에 이르면 경고**, **100%에 이르면 원격 AI 요청을 차단**합니다.

```text
Monthly AI budget limit reached. Remote AI requests are blocked.
Switch to local AI mode or increase the configured budget.
```

비용이 정확히 제공되지 않는 경우 "예상(estimated)"으로 표시됩니다. 로컬 AI는 비용 0으로 표시됩니다.

## 6. 자주 묻는 질문 (FAQ)

**Q. AI가 명령을 자동으로 실행하나요?**
A. 아니요. 기본적으로 절대 자동 실행하지 않습니다. 항상 보여 주고, 사용자가 Enter로 허락해야 실행됩니다.

**Q. AI 없이 그냥 터미널로만 쓸 수 있나요?**
A. 네. `ai`를 부르지 않으면 일반 리눅스 터미널과 동일합니다.

**Q. 내 비밀번호나 API 키가 외부로 새나요?**
A. 민감 정보는 전송 전 자동으로 가려지며, 가려지지 못하면 원격 전송이 차단됩니다. `paranoid` 프로파일은 아예 원격 전송을 막습니다.

**Q. AI가 틀린 명령을 만들면요?**
A. 실행 전 위험도와 설명을 읽고 거부할 수 있습니다. 또 존재하지 않는 프로그램명을 만들면 경고합니다.

```text
Generated command: gitx log --oneline
Warning: 'gitx' not found on this system.
Did you mean 'git'? [a]ccept suggestion  [e]dit  [c]ancel
```

**Q. 실수로 파일을 바꿨어요. 되돌릴 수 있나요?**
A. 파일 수정 등 일부 작업은 `ai undo last`로 되돌릴 수 있습니다(§4.2). 단 삭제·전송 등은 불가합니다.

**Q. 과거에 했던 작업을 찾고 싶어요.**
A. `ai history "nginx 500 관련"` 또는 `ai recall "어제 배포 작업"`으로 자연어 검색이 됩니다.

**Q. 화면이 어떻게 생겼는지 보고 싶어요.**
A. `→ 06_화면_기능_정의서.md 참조`.

## 7. 문제 해결

### 7.1 AI 응답이 없거나 느립니다 (타임아웃)

AI 서비스에 문제가 생겨도 **터미널은 계속 정상 동작**합니다. 타임아웃 기준은 연결 5초 / 첫 응답 15초 / 전체 60초 / 긴 요약 180초입니다.

```text
AI request timed out. Shell mode remains active.
AI service unavailable. You can continue using the terminal normally.
```

→ 잠시 후 다시 시도하거나, 일반 명령으로 작업을 계속하세요. `ai doctor`로 AI 연결 상태를 점검할 수 있습니다.

### 7.2 셸 통합(hook)이 설치되지 않았습니다

`cd`나 브랜치 변경이 AI에 늦게 반영되면 통합이 안 됐을 수 있습니다.

```bash
ai init shell --dry-run   # 무엇이 추가될지 확인
ai init shell             # 동의 후 설치
```

hook이 실패해도 일반 셸 사용은 중단되지 않습니다. 안 되면 자동으로 Wrapper 방식으로 대체됩니다.

### 7.3 마스킹 때문에 원격 AI가 막혔습니다

민감 정보가 안전하게 가려지지 못하면, 보호를 위해 원격 전송을 막습니다(fail-closed).

```text
Sensitive data may be present in the selected context.
Remote AI request blocked by privacy policy.
Use local AI mode or reduce context scope.
```

→ 보낼 컨텍스트 범위를 줄이거나, 로컬 AI 모드로 전환하세요.

### 7.4 같은 명령인데 결과가 달라요

AI 특성상 결과가 조금씩 달라질 수 있습니다. 명령 생성은 가급적 일관되게 동작하도록 설정돼 있으며, 동일 결과를 다시 쓴 경우 `(cached)` 배지가 붙습니다. 위험도 점수는 항상 동일 규칙으로 계산되므로 같은 명령은 같은 위험도를 받습니다.

### 7.5 `ai` 명령이 이미 있다고 나옵니다

다른 프로그램과 이름이 겹친 경우입니다. 안내에 따라 `aish` 같은 대체 이름을 쓰면 됩니다(§1.1).

### 7.6 WSL/macOS에서 일부 안전 기능이 제한됩니다

플랫폼에 따라 실행 중 감시 같은 일부 기능이 제한될 수 있습니다. 이때는 High 이상 명령의 확인이 더 강화됩니다.

```bash
ai doctor --guardrails    # 현재 플랫폼에서 가능한 안전 기능 확인
```
