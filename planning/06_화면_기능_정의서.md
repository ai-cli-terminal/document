# 06. 화면 기능 정의서
> **프로젝트명**: AI CLI 통합 리눅스 터미널
> **버전**: v1.0
> **작성일**: 2026-06-01
> **기술 스택**: Rust · ratatui · tokio · portable-pty · SQLite (대안: Go)
---

> 본 프로젝트는 GUI 웹 애플리케이션이 아니라 **터미널/TUI**다. 따라서 여기서 말하는 "화면"은 웹페이지가 아니라
> **터미널 뷰 · 패널 · 입력 모드**를 의미한다. 모든 목업은 실제 터미널 출력을 ASCII로 표현한다.
> UI 모드는 Shell Mode(기본) / AI Inline(`ai "..."`) / AI Dedicated(`ai shell`) / Hybrid(`!cmd`, `/cmd`)로 구분된다.
> → 컴포넌트·UI 모드 원본은 `01-core-design.md` §6.1, 아키텍처는 `00-overview-architecture.md` §5 참조.

## 2. 주요 화면 상세

핵심 5개 화면을 상세 정의한다: ① Visual Command Preview Pane, ② Diff 미리보기, ③ 컨센트·전송 범위 표시, ④ Hybrid shell, ⑤ ai doctor.
각 화면은 **ASCII 목업 + 기능 설명 + 상태/이벤트 + 관련 FR/명령 상호참조**로 구성한다.

### 2.1 SCR-T-PREVIEW-001 — Visual Command Preview Pane

AI가 명령어 또는 파일 변경을 제안할 때, 실행 전 정보를 별도 패널에 시각적으로 표시한다.
좌측 Terminal 패널(사용자 입력·출력)과 우측 Preview 패널(제안 상세)로 분할된다.
표시 항목: 생성된 명령어, 위험도, 예상 변경 파일 목록, diff preview, dry-run 결과, 토큰 사용량, 예상 비용, 실행 컨텍스트, 정책 프로파일, 요구 권한.

#### 화면 구성 (ASCII 목업)

```text
+---------------- Terminal ----------------+----------------- Preview ------------------+
| [local:/home/user/app]$                  |  Command                                   |
| $ ai run "console.log 제거해줘"           |    sed -i '/console.log/d' app.js          |
|                                           |  ----------------------------------------- |
| (waiting for confirmation...)             |  Risk        : Medium (score 38/100)       |
|                                           |  Files        : 1  (app.js)                |
|                                           |  Context      : local : /home/user/app     |
|                                           |  Profile      : balanced                   |
|                                           |  Permission   : FileWrite                  |
|                                           |  Matched skill: (none)   Score: -          |
|                                           |  ----------------------------------------- |
|                                           |  Diff preview                              |
|                                           |    - console.log("debug")                  |
|                                           |    + (removed)                             |
|                                           |  ----------------------------------------- |
|                                           |  Tokens : 3,240 in / 620 out               |
|                                           |  Est.cost: $0.0041                          |
|                                           |  ----------------------------------------- |
|                                           |  Apply changes? [y/N]   (e: edit, Ctrl+C)  |
+-------------------------------------------+--------------------------------------------+
```

#### 기능 설명

- AI 경로(`ai cmd`/`ai run`/`ai run --preview`/`ai preview`)에서 생성된 명령을 실행하기 전 우측 Preview 패널에 요약한다.
- 위험도는 0~100 스케일로 표기하고 등급(Low 0~24 / Med 25~49 / High 50~79 / Critical 80~100)을 병기한다.
- 요구 권한(ReadOnly < FileWrite < ProcessControl < Network < Privileged < Destructive)과 현재 정책 프로파일(balanced/paranoid 등)을 함께 표시한다.
- 파일 변경 명령은 영향 파일 목록과 diff preview를 포함한다(상세는 SCR-T-PREVIEW-002).
- 매칭된 스킬(§26)이 있으면 스킬명과 매칭 점수를 표시해 투명성을 확보한다.
- 토큰 사용량·예상 비용을 표시해 비용 인식을 돕는다(SCR-T-USAGE-001과 연동).
- `balanced` 프로파일에서는 Medium 이상, `paranoid`에서는 모든 AI 명령에 대해 본 패널 확인을 강제한다.

#### 상태/이벤트

| 상태 | 트리거 | UI 동작 |
|---|---|---|
| `idle` | AI 응답 대기 전 | Preview 패널 비표시 |
| `streaming` | 명령 생성 중 | 부분 결과 표시, **완성 전 실행 불가** |
| `ready` | 명령 생성 완료 | 위험도·diff·비용 표시, `Apply? [y/N]` 노출 |
| `confirmed` | 사용자 `y` 입력 | PTY로 전달 → 실행 → 감사 로그 기록 |
| `edited` | 사용자 `e` 입력 | 멀티라인 편집 모드 진입 |
| `cancelled` | `Ctrl+C` | AI 요청만 취소, 쉘 프롬프트로 복귀(세션 로그 `cancelled`) |

#### 관련 FR / 명령 / 상호참조

- 관련 FR: `FR-AI-005`(preview 패널), `FR-AI-004`(위험도 표기), `FR-GUARD-001`(실행 전 정책 검사), `FR-OPS-002`(토큰·비용 표시).
- 관련 명령: `ai run`, `ai run --preview`, `ai preview`, `ai cmd`.
- → 원본 `01-core-design.md` §6.1 Visual Command Preview Pane, §9.3 `ai run` 실행 전 표시 참조.
- → 위험도 등급: `02-security-policy.md` §10.1 참조.

### 2.2 SCR-T-PREVIEW-002 — Diff 미리보기 (dry-run / diff)

파일 변경 명령(`sed -i`, `perl -pi`, `awk` 재작성, formatter, codemod, 대량 rename, AI 생성 스크립트 등)을 실행하기 전,
임시 복사본에서 실행하거나 도구 자체의 dry-run을 사용해 변경 결과를 diff로 보여준다.

#### 화면 구성 (ASCII 목업)

```text
+------------------------- Diff Preview -------------------------+
| Proposed command:                                             |
|   sed -i 's/foo/bar/g' config.txt                             |
|---------------------------------------------------------------|
| Mode : dry-run on temporary copy (/tmp/ai-terminal-preview)   |
|                                                               |
|   --- config.txt                                              |
|   +++ config.txt                                              |
|   - foo_enabled=true                                          |
|   + bar_enabled=true                                          |
|   - foo_path=/opt/foo                                         |
|   + bar_path=/opt/foo                                         |
|---------------------------------------------------------------|
| Risk level : Medium      Files affected : 1                   |
| Apply changes? [y/N]                                          |
+---------------------------------------------------------------+
```

dry-run을 직접 지원하는 도구는 우선 그 옵션을 사용한다.

```text
+--------------- Dry-run Support Matrix ---------------+
| rsync     -> --dry-run                               |
| git clean -> -n                                      |
| terraform -> plan                                    |
| kubectl   -> --dry-run=client                        |
| npm       -> (일부 명령) --dry-run                   |
| rm        -> 직접 없음 -> 대상 파일 목록 미리 출력   |
+------------------------------------------------------+
```

#### 기능 설명

- AI는 destructive 명령보다 dry-run 대안을 **먼저** 제안해야 한다.
- 명령 자체가 dry-run을 지원하면 임시 복사본 실행 대신 도구의 dry-run을 우선 사용한다.
- `rm`처럼 dry-run이 없는 경우 삭제 대상 파일 목록을 먼저 출력한다(대량 작업 보호).
- 심볼릭 링크를 따라가는 명령은 별도 경고를 표시하고, 대상 경로를 realpath로 정규화해 홈/루트/시스템 디렉터리 확장 여부를 검사한다.
- `[preview]` 설정: `force_preview_for_file_modification = true`, `use_dry_run_when_available = true`.

#### 상태/이벤트

| 상태 | 트리거 | UI 동작 |
|---|---|---|
| `computing` | 임시 복사본 dry-run 수행 중 | "Computing diff..." 표시 |
| `diff_ready` | diff 계산 완료 | diff 본문 + `Apply? [y/N]` |
| `no_change` | 변경 없음 | "No changes" 표시 후 종료 |
| `symlink_warn` | 심볼릭 링크 감지 | 추가 경고 라인 노출 |
| `applied` | `y` 입력 | 실제 경로에 적용 → before/after 스냅샷 기록 |

#### 관련 FR / 명령 / 상호참조

- 관련 FR: `FR-AI-006`(diff/dry-run 우선 제안), `FR-GUARD-004`(경로 정규화·심볼릭 링크 검사), `FR-STORE-003`(before/after 스냅샷).
- 관련 명령: `ai preview`, `ai run --preview`.
- → 원본 `01-core-design.md` §9.4 Dry-run/Diff 미리보기, §19.4 심볼릭 링크·경로 우회 참조.
- → 사용자 시나리오 `00-overview-architecture.md` §4.5 참조.

### 2.3 SCR-T-PREVIEW-003 — 컨센트·전송 범위 표시 (Zero-Trust)

원격 AI 호출 전, 모델 프로바이더로 전송될 컨텍스트의 **범위와 마스킹 결과**를 사용자에게 보여주고 동의를 받는다.
Zero-Trust Context Pipeline의 Consent & Scope Display 단계에 해당한다.

#### 화면 구성 (ASCII 목업)

```text
+----------------- Remote AI · Context Consent -----------------+
| The following context will be sent to the remote AI provider: |
|                                                               |
|   [+] User request                                            |
|   [+] Current directory path   (/home/user/app)               |
|   [+] Last command and exit code  (ls logs / 0)               |
|   [+] 120 lines of stderr                                     |
|   [+] Git branch name  (main)                                 |
|   [+] package.json scripts                                    |
|---------------------------------------------------------------|
| Masked before sending:                                        |
|   - 3 email addresses    -> [EMAIL_REDACTED]                  |
|   - 1 API token          -> [TOKEN_REDACTED]                  |
|   - 2 IP addresses       -> [IP_REDACTED]                     |
|---------------------------------------------------------------|
| Provider : remote_primary    Profile : balanced              |
| Send to remote AI? [Y/n]      (l: use local AI instead)       |
+---------------------------------------------------------------+
```

마스킹 실패 시(민감 데이터 잔존 가능) 원격 호출을 차단한다.

```text
+------------------- Privacy Policy Block ----------------------+
| Sensitive data may be present in the selected context.        |
| Remote AI request blocked by privacy policy.                  |
| Use local AI mode or reduce context scope.                    |
+---------------------------------------------------------------+
```

#### 기능 설명

- 전송 항목은 Data Minimization 원칙에 따라 최소화한다(전체 로그 대신 에러 주변 N라인, 전체 `.env` 대신 키 이름만 등).
- Secret/PII 마스킹은 원격 호출 전 필수이며, 마스킹된 항목 수를 명시한다.
- `balanced`는 민감 정보 포함 시(`when_sensitive`)에만 본 화면을 노출하고, `paranoid`는 **항상** 노출한다.
- 마스킹 실패 감지 시 fail-closed: 원격 호출을 중단하고 로컬 AI 또는 범위 축소를 안내한다.
- 조직 정책(`block_remote_ai = true`)이 적용되면 본 화면 없이 로컬 호출로 강제될 수 있다.

#### 상태/이벤트

| 상태 | 트리거 | UI 동작 |
|---|---|---|
| `scope_ready` | 컨텍스트 정제·마스킹 완료 | 전송 항목·마스킹 결과 표시 |
| `consented` | `Y` 입력 | 원격 AI 호출 진행 |
| `local_fallback` | `l` 입력 | 로컬 AI 모드로 전환 |
| `masking_failed` | 마스킹 실패 감지 | 원격 호출 차단, 안내 표시 |
| `org_blocked` | 조직 정책 차단 | 로컬 강제, 사용자 완화 불가 |

#### 관련 FR / 명령 / 상호참조

- 관련 FR: `FR-SEC-002`(Secret/PII 마스킹), `FR-SEC-003`(Data Minimization·컨센트), `FR-SEC-006`(마스킹 실패 시 fail-closed), `FR-CTX-004`(전송 범위 표시).
- 관련 명령: AI 경로 전반(`ai ask`, `ai cmd`, `ai fix`, `ai summarize`), `/context`(Hybrid 내부).
- → 원본 `02-security-policy.md` §10.3 Zero-Trust Context Pipeline, §10.4 마스킹 참조.
- → 설계 원칙 `00-overview-architecture.md` §3 원칙 6~9 참조.

### 2.4 SCR-T-HYBRID-001 — AI Dedicated / Hybrid shell (`ai shell`)

`ai shell`로 진입하는 대화형 전용 모드다. 자연어 AI 요청과 일반 쉘 명령(`!cmd`), 내부 제어 명령(`/cmd`)을 한 프롬프트에서 혼합한다.

#### 화면 구성 (ASCII 목업)

```text
+------------------------- AI Dedicated / Hybrid Shell ----------------------+
| $ ai shell                                                                 |
| Entering AI shell. Type /exit to leave, Ctrl+D to quit.                    |
|                                                                            |
| ai> nginx 500 에러 원인 분석해줘                  <- 자연어 -> AI 요청     |
|     [AI] error.log를 확인하겠습니다. 최근 로그를 보여주세요.               |
|                                                                            |
| ai> !tail -n 100 /var/log/nginx/error.log         <- !cmd -> 즉시 실행     |
|     2026/06/01 11:20:01 [error] upstream timed out ...                     |
|                                                                            |
| ai> 이 로그 기준으로 원인 요약해줘                                          |
|     [AI] upstream(앱 서버) 응답 지연으로 502/500이 발생했습니다.            |
|         제안: proxy_read_timeout 상향 또는 앱 서버 헬스체크 확인.           |
|                                                                            |
| ai> /status                                       <- /cmd -> 내부 제어     |
|     context: [ssh:prod-web-01:/var/www/app]  profile: balanced            |
|     last cmd: tail (exit 0)   tokens: 5,120 in / 880 out                   |
|                                                                            |
| ai> /exit                                                                  |
| $                                                                          |
+----------------------------------------------------------------------------+
```

입력 규칙과 내부 제어 명령:

```text
+--------------- Hybrid Mode 입력 규칙 ---------------+
| 일반 자연어    -> AI 요청으로 처리                  |
| !command       -> 일반 쉘 명령으로 즉시 실행        |
| /command       -> AI shell 내부 제어 명령           |
| Ctrl+C         -> 현재 AI 요청/실행 중 명령 취소    |
| Ctrl+D         -> AI shell 종료                     |
+----------------------------------------------------+
| 내부 제어 명령:                                     |
|   /status  컨텍스트 표시                            |
|   /context 전송 범위 표시                           |
|   /shell   쉘 모드 전환                             |
|   /clear   대화 초기화                              |
|   /policy  정책 표시                                |
|   /usage   토큰/비용                                |
|   /exit    종료                                     |
+----------------------------------------------------+
```

#### 기능 설명

- 자연어는 AI 요청으로, `!`는 즉시 쉘 실행으로, `/`는 내부 제어로 분기 처리한다.
- `!cmd`로 실행된 일반 명령도 Policy Engine·위험도 분류·Guardrails 감시를 동일하게 받는다.
- AI 요청은 취소 가능한 비동기 단위로 관리되어, 응답 대기 중 `Ctrl+C`가 터미널을 종료하지 않고 요청만 취소한다.
- 비밀번호 입력 프롬프트 감지 시 AI 컨텍스트 수집을 중단한다(§19.2).
- 인터랙티브 명령(`vim`, `top`, `less` 등)은 PTY raw mode로 처리하고 전체 화면 버퍼는 AI 컨텍스트에서 제외한다.

#### 상태/이벤트

| 상태 | 트리거 | UI 동작 |
|---|---|---|
| `nl_request` | 자연어 입력 | AI 요청 → 스트리밍 응답 |
| `shell_exec` | `!cmd` 입력 | 정책 검사 후 PTY 즉시 실행 |
| `internal` | `/cmd` 입력 | 내부 핸들러 응답(`/status` 등) |
| `ai_cancel` | `Ctrl+C` | AI 요청만 취소, 프롬프트 복귀 |
| `exit` | `/exit` 또는 `Ctrl+D` | Shell Mode로 복귀 |

#### 관련 FR / 명령 / 상호참조

- 관련 FR: `FR-SHELL-002`(UI 모드 분기), `FR-SHELL-005`(Hybrid 입력 규칙), `FR-AI-009`(AI 요청 인터럽트), `FR-CTX-001`(컨텍스트 동기화).
- 관련 명령: `ai shell`, `!<command>`, `/status`, `/context`, `/policy`, `/usage`, `/exit`.
- → 원본 `01-core-design.md` §6.1 UI 모드·Hybrid Mode 입력 규칙·AI 요청 인터럽트 참조.

### 2.5 SCR-T-DOCTOR-001 — `ai doctor` 진단 뷰

시스템 구성 요소(쉘·PTY·AI 프로바이더·샌드박스·정책·마스킹·컨텍스트)의 상태를 점검해 리포트로 보여준다.

#### 화면 구성 (ASCII 목업)

```text
+-------------------------- AI Terminal Doctor -------------------------+
|  $ ai doctor                                                          |
|                                                                       |
|  Shell        : bash 5.2 detected                          [ OK ]     |
|  PTY          : pseudo-terminal allocated                  [ OK ]     |
|  AI provider  : remote provider reachable                  [ OK ]     |
|  Sandbox      : container backend available                [ OK ]     |
|                 user namespace enabled / no-new-privileges [ OK ]     |
|  Policy       : active profile: balanced                   [ OK ]     |
|  Masking      : secret rules 3 loaded / pii rules 3 loaded [ OK ]     |
|  Context      : cwd synchronized / git branch synchronized [ OK ]     |
|-----------------------------------------------------------------------|
|  Options: --ai  --sandbox  --policy  --context  --mcp                 |
|           --export-debug-bundle                                       |
+-----------------------------------------------------------------------+
```

특정 영역만 점검할 수 있다.

```text
+----------------- ai doctor --sandbox -----------------+
| Sandbox backend     : container          [ OK ]       |
| User namespace       : enabled            [ OK ]       |
| Drop capabilities    : cap_drop ALL       [ OK ]       |
| no-new-privileges    : applied            [ OK ]       |
| Network              : none               [ OK ]       |
| Read-only rootfs     : true               [ OK ]       |
| Limits               : mem 512m / cpu 1.0 / pids 128  |
+-------------------------------------------------------+
```

#### 기능 설명

- `ai doctor [--ai] [--sandbox] [--policy] [--context] [--mcp]`로 영역별 진단을 수행한다.
- `--export-debug-bundle`은 디버그 번들을 생성한다.
  - **포함**: 설정 파일, 정책 프로파일, 최근 에러 로그, AI provider 상태, 샌드박스 상태, PTY 상태, 마스킹된 세션 이벤트 일부.
  - **제외**: Secret, PII, 전체 명령 히스토리, 원문 로그, `.env`, private key, credential 파일.
- 마스킹 룰 적재 수(secret 3 / pii 3, §31.8 baseline)를 표시해 마스킹 파이프라인 동작을 확인한다.
- 컨텍스트 동기화(cwd·git branch) 상태를 점검해 Context Consistency Manager 상태를 노출한다.

#### 상태/이벤트

| 상태 | 트리거 | UI 동작 |
|---|---|---|
| `running` | `ai doctor` 실행 | 항목별 점검 진행 표시 |
| `ok` | 모든 항목 정상 | `[ OK ]` 배지 |
| `warn` | 일부 항목 경고 | `[WARN]` + 사유 |
| `fail` | 항목 실패 | `[FAIL]` + 조치 안내 |
| `bundle_exported` | `--export-debug-bundle` | 번들 경로 표시(민감 정보 제외 확인) |

#### 관련 FR / 명령 / 상호참조

- 관련 FR: `FR-OPS-003`(진단 명령), `FR-OPS-004`(디버그 번들 민감정보 제외), `FR-SEC-002`(마스킹 룰 적재 확인), `FR-CTX-001`(컨텍스트 동기화 점검).
- 관련 명령: `ai doctor`, `ai doctor --sandbox`, `ai doctor --export-debug-bundle`.
- → 원본 `04-config-ops-testing.md` §23.1 진단 명령, §23.2 디버그 번들 참조.

## 3. 부가 화면 요약

상세 5종 외 인벤토리 화면의 핵심 동작을 간단히 정리한다(목업 일부 포함).

### 3.1 SCR-T-SHELL-002 — 컨텍스트 스택 프롬프트

실행 위치가 바뀌면 프롬프트가 현재 컨텍스트를 명시한다.

```text
[local:/home/user/project]$
[ssh:prod-web-01:/var/www/app]$
[container:app:/usr/src/app]$
[k8s:prod/default/api-pod:/app]$
```

→ 관련 FR: `FR-CTX-002`(컨텍스트 스택), `FR-CTX-005`(로컬/원격 구분). 원본 `01-core-design.md` §7.5.

### 3.2 SCR-T-SHELL-003 — 컨텍스트 불일치 감지·동기화

```text
Context mismatch detected.
AI context cwd:   /home/user/project
Actual shell cwd: /home/user/project/backend
Refresh AI context before generating command? [Y/n]
```

→ 관련 FR: `FR-CTX-001`(상시 동기화), `FR-CTX-003`(built-in 명령 동기화). 원본 §7.6.

### 3.3 SCR-T-AISUG-002 — `ai run` 실행 전 위험도·확인

```text
Generated command: rm -rf ./node_modules
Risk level: Medium
Reason: Deletes a large dependency directory, but path is scoped.
Execute? [y/N]
```

→ 관련 FR: `FR-AI-003`(명령 생성), `FR-AI-004`(위험도 분석), `FR-SEC-001`(실행 전 확인). 원본 §9.3.

### 3.4 SCR-T-AISUG-003 — `ai fix` / 자가 치유 재제안

```text
The AI-generated command failed.
Cause: The target directory does not exist.
Previous: tar -czf backup.tar.gz ./data
Suggested fix: mkdir -p ./data && tar -czf backup.tar.gz ./data
Risk: Medium — Creates a directory and archives it.
Run suggested fix? [y/N]
```

자동 재실행은 항상 금지(1회 자동 *분석·제안*만). High 이상·sudo·삭제·네트워크 전송·예산 초과 시 자가 치유 비활성화.
→ 관련 FR: `FR-AI-007`(에러 분석), `FR-AI-008`(자가 치유 — 자동 실행 금지). 원본 `04-config-ops-testing.md` §16.3.

### 3.5 SCR-T-POLICY-001 — 정책 프로파일 표시/전환

```text
$ ai policy show
Active profile : balanced
  confirm_level         : medium_and_above
  block_remote_ai       : false
  mask_secrets / mask_pii: true / true
  auto_execute          : false
Available : paranoid | balanced | poweruser(Phase2) | dev(Phase2)
$ ai policy set paranoid
Profile changed to 'paranoid'. All AI commands now require confirmation.
```

→ 관련 FR: `FR-SEC-004`(정책 프로파일 전환), `FR-SEC-005`(조직 정책 우선). 원본 `02-security-policy.md` §12.2.

### 3.6 SCR-T-POLICY-002 — Guardrail 발동 알림

실행 중 동적 감시(Execution Guardrails Engine)가 비정상 행위를 감지하면 일시정지 또는 차단한다.

```text
Execution paused. The command is attempting to delete more than 1,000 files.
Command: rm -rf ./cache   Affected path: ./cache   Continue? [y/N]
---
Command terminated by guardrails.
Reason: Attempted to access /var/run/docker.sock; current policy profile blocks Docker socket access.
```

→ 관련 FR: `FR-GUARD-002`(실행 중 동적 감시), `FR-GUARD-003`(프로세스 그룹 종료). 원본 `01-core-design.md` §8.

### 3.7 SCR-T-USAGE-001 — 토큰·비용·예산

```text
AI usage: 3,240 input / 620 output tokens   Estimated cost: $0.0041
Session total: $0.083 / $2.00   Monthly total: $12.42 / $30.00
---
Monthly AI budget limit reached. Remote AI requests are blocked.
Switch to local AI mode or increase the configured budget.
```

예산 80% 경고 / 100% 차단. → 관련 FR: `FR-OPS-002`(토큰·비용·예산). 원본 `04-config-ops-testing.md` §13.

### 3.8 SCR-T-AISUG-001 — AI 제안 패널 / 실시간 스트리밍

긴 명령/스크립트 생성 중 중간 결과를 스트리밍한다. **스트리밍 중인 명령은 완성 전까지 실행할 수 없다.**

```text
Generating command...
[partial] find . -type f -name "*.log" ...
---
Generated command:
find . -type f -name "*.log" -mtime +7 -print
Risk: Low   Run? [y/N]
```

→ 관련 FR: `FR-AI-002`(스트리밍 제안), `FR-AI-005`(preview). 원본 `01-core-design.md` §6.1 실시간 스트리밍 제안.

### 3.9 SCR-T-REMOTE-001 — 원격 승인·페어링 (Phase 3)

외출 중 AI 확인 프롬프트를 모바일/웹으로 받아 승인/거부/편집한다.

```text
+----------- AI Terminal · Remote Approval (mobile) -----------+
| Device: pixel-8 (approve)        Session: prod-web-01        |
|-------------------------------------------------------------|
| AI requests confirmation:                                   |
|   Command : rm ./logs/app.2026-05-*.log                     |
|   Risk    : Medium (score 30/100)                           |
|   Context : ssh:prod-web-01 : /var/www/app                  |
|-------------------------------------------------------------|
|   [Approve]   [Reject]   [Edit]      (biometric required)   |
+-------------------------------------------------------------+
```

원격 승인 가능 범위는 정본(`→ 05 §30-13`)을 따른다: 모바일에서 승인 가능한 건 **Low/Medium까지(opt-in, Medium은 강한 확인)**이고, **High·Critical 명령과 외부 부작용 MCP 도구의 원격 승인은 기본 차단**(로컬 터미널에서 승인하도록 안내). 따라서 위 목업도 Medium 명령 예시다 — High(예: `systemctl restart nginx`, 62/100)가 오면 [Approve]가 비활성화되고 "로컬에서 승인하세요" 안내가 뜬다. 출력 스트리밍에도 Secret/PII 마스킹 적용.
→ 관련 FR: `FR-REMOTE-001`(원격 모니터링·알림), `FR-REMOTE-002`(원격 승인·High/Critical 차단), `FR-REMOTE-003`(E2E·페어링). 원본 `03-subsystems-skill-mcp-remote.md` §28.4, `05-roadmap-enhancements-decisions.md` §30-13.
